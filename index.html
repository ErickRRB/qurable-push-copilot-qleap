POST /v2/activation/targets POST /v2/activation/jobs GET /v2/activation/jobs/{id}/executions
EXISTS · Manager v2

Dashboards + Widgets

Crear, ordenar, validar y ejecutar widgets sobre queries DSL o almacenadas.

POST /v2/console/insights/dashboards POST /v2/console/insights/widgets POST /v2/console/insights/widgets/validate
EXISTS · Qurable API

Eventos

Eventos síncronos y asíncronos, tipos custom y datos para construir medición.

POST /v2/events/track POST /v2/events/track-async POST /v2/custom-event-types

Certeza: son contratos publicados en API v2. El MCP documentado cubre reporting; las operaciones transaccionales requieren clientes API con scopes. Acceso, modelo semántico, stored queries y permisos deben validarse en el tenant del piloto.
